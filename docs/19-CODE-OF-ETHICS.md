## 📋 Códigos de Ética

### 1. TRANSPARÊNCIA ABSOLUTA

#### Sobre Preços

```
✅ CORRETO:
├─ "Comissão: 25% em todas vendas"
├─ "Preço final: R$ 99 + R$ 24.75 taxa"
└─ "Breakdown claro de custos"

❌ INCORRETO:
├─ "Software a partir de R$ 99" (sem taxa oculta)
├─ "Melhor preço do mercado" (sem fonte)
└─ Esconder custos adicionais
```

**Regra:** Todo cliente sabe EXATAMENTE o que paga e por quê, ANTES de comprar.

#### Sobre Produto

```
✅ CORRETO:
├─ "Limitado a 100 clientes/mês"
├─ "Atualizado há 3 meses"
├─ "Suporte em horário comercial"
└─ "Funciona melhor em Chrome"

❌ INCORRETO:
├─ Listar features que não têm
├─ Esconder limitações
├─ Prometer suporte 24/7 se não temos
└─ Fake testimonials
```

**Regra:** Toda feature é verificável. Toda limitação é explícita.

#### Sobre Dados

```
✅ CORRETO:
├─ "Coletamos: email, empresa, estado"
├─ "Razão: contato + analytics"
├─ "Compartilhamos: NUNCA com 3º"
├─ "Deletamos em: 90 dias ou sob request"
└─ Policy a clique

❌ INCORRETO:
├─ Vender dados a publicidade
├─ Compartilhar sem consentimento
├─ "Privacy policy" em 50 páginas ilegível
└─ Reter dados após cancelamento
```

**Regra:** Dados são sagrados. Transparência máxima.

### 2. JUSTIÇA COMERCIAL

#### Com Fornecedores (Desenvolvedoras de software)

```
PRINCÍPIO: Parceria ganha-ganha. Nunca lesar.

✅ CORRETO:
├─ Pagar comissão acordada 100% no prazo
├─ Transparente sobre metrics
├─ Sugestões de melhoria (não ordem)
├─ Respeitar propriedade intelectual
├─ Comunicação regular
└─ Aumentar comissão se crescer juntos

❌ INCORRETO:
├─ Atrasar pagamento "cash flow tight"
├─ Mentir sobre demanda
├─ Forçar features grátis
├─ Negociar mal porque pequeno
├─ Usar código sem crédito
└─ Cobrar taxa "oculta" de promoção
```

**Regra:** Fornecedor confia em nós. Nunca quebrar essa confiança.

#### Com Clientes (PMEs)

```
PRINCÍPIO: Ajudar negócio deles crescer. Nunca sacanear.

✅ CORRETO:
├─ Assessoria honesta: é bom pra você?
├─ Mostrar alternativas mesmo que perder
├─ Suporte rápido
├─ Resolvemos problemas grátis 1px
├─ Não cobrar por "migração"
└─ Dar time pra decidir (sem pression)

❌ INCORRETO:
├─ "É a melhor solução" (pra eles ou pra nós?)
├─ Hidden fees (setup, suporte, etc)
├─ Vender enterprise pra micro
├─ Contrato com lock-in 2 anos
├─ Fazer duro pra cancelar
└─ SLA impossível de cumprir
```

**Regra:** Cliente sucesso = nosso sucesso.

#### Com Parceiros (Contadores, consultores)

```
PRINCÍPIO: Respeitamos expertise. Comissão justa.

✅ CORRETO:
├─ Pagar 15-20% referral
├─ Lead scoring transparente
├─ Contatos com respeito
├─ Apoio materiais marketing
├─ Escalação se problema
└─ Reconhecer em públic

❌ INCORRETO:
├─ Pagar 5% "porque é pouco"
├─ Roubar cliente do parceiro
├─ Fazer contato direto se parceiro referiu
├─ Não pagar se "conversão baixa"
├─ Usar relacionamento deles pra próprio uso
└─ Cobrar comissão deles pra "lead marketing"
```

**Regra:** Parceiros são aliados, não vassalos.

### 3. INTEGRIDADE PESSOAL

#### Conflito de Interesse

```
Cenário 1: CEO tem startup concorrente
├─ AÇÃO: Comunicar conselho
├─ DECISÃO: Vender stake ou sair da empresa
└─ NUNCA: Esconder + usar recursos TupãSoft

Cenário 2: Dev recebe "presente" de fornecedor
├─ AÇÃO: Reportar imediatamente
├─ VALOR > R$500: Recusar ou devolver
└─ NUNCA: Aceitar pra dar preferência

Cenário 3: Cliente oferece propina pra desconto
├─ AÇÃO: Recusar + documentar
├─ EFEITO: Denunciar à polícia se > R$1k
└─ NUNCA: Aceitar por qualquer motivo
```

**Regra:** Conflito de interesse = admissão + resolução imediata.

#### Sigilo Comercial

```
✅ PROTEGEMOS:
├─ Segredos de negócio (formação preço)
├─ Dados clientes (nomes, faturamento)
├─ Estratégia (roadmap não publicado)
└─ Relacionamentos comerciais

❌ REVELAMOS:
├─ À polícia (mandado)
├─ Para pesquisa acadêmica (anonimizado)
├─ Por lei (LGPD, fiscalização)
└─ Públic financesamento (IPO)

NUNCA:
├─ Vazamos propositalmente
├─ Vendemos a concorrente
├─ Usamos pra vantagem pessoal
└─ Compartilhamos sem NDA
```

**Regra:** Confiança vale ouro.

#### Respeito ao Tempo & Recursos

```
Tempo da empresa:
├─ Trabalho pessoal: máx 5% da jornada/semana
├─ Rede social: recreio (30min)
├─ Não trabalhar dormindo
└─ Férias realmente férias

Recursos da empresa:
├─ Laptop: trabalho APENAS
├─ Internet: trabalho APENAS
├─ Não usar para stream pessoal
├─ Reembolso necessário se uso pessoal
└─ Não instalar software pessoal
```

**Regra:** Respeitar o que nos é emprestado.

### 4. COMPLIANCE LEGAL

#### Cumprimento de Leis

```
🇧🇷 BRASIL (Obrigatório):
├─ Lei 12.965/2014 (Marco Civil)
├─ Lei 13.718/2018 (Crimes cibernéticos)
├─ Lei 13.709/2018 (LGPD)
├─ Código de Defesa do Consumidor
├─ CLT (Direitos trabalhistas)
└─ COAF (Lavagem dinheiro)

🌐 INTERNACIONAL:
├─ GDPR (EU citizens)
├─ CCPA (California)
├─ PCI-DSS (pagamentos)
└─ Embargos comerciais

Não aceitamos:
├─ "Ninguém vai saber"
├─ "Lei é pra grandes"
├─ "Multa é menor que lucro"
├─ "Político fez pior"
└─ Desculpas em geral
```

**Regra:** Lei é lei. Sem exceção.

#### Anti-Corrupção

```
PROIBIDO em qualquer negociação:
├─ Propina (dinheiro, presente > R$200)
├─ Favorecimento (contrato sem licitação)
├─ Facilitação (pagar pra agilizar processo)
├─ Nepotismo (contratar por relação)
└─ Fraude fiscal

OBRIGATÓRIO:
├─ Licitação pública se gasto > R$50k
├─ Documentação de compras (NF)
├─ Transparência parceiros
├─ Denunciar suspeita
└─ Audit anual
```

**Regra:** Zero-tolerance no sigilo.

#### Proteção de Dados (LGPD)

```
Dados são responsabilidade NOSSA:
├─ Segurança: criptografia, backups
├─ Consentimento: "opt-in" explícito
├─ Transparência: policy clara
├─ Direitos: deletar, corrigir, portar
├─ Retenção: 90d sem usar
├─ Breach: notificar em 72h
└─ DPO: responsável designado
```

**Regra:** LGPD não é chato, é essencial.

## ⚖️ Dilemas Éticos: Como Decidir?

### Framework de Decisão

Quando incerto, pergunte:

```
1. É legal?
   └─ Não? → NÃO FAÇA

2. É justo com todas as partes?
   ├─ Desfavorecendo alguém? → Reconsiderar
   └─ Win-win? → OK continuar

3. Eu explicaria ao meu avó?
   ├─ Constrangido? → NÃO FAÇA
   └─ Orgulho? → OK fazer

4. E se todos fizessem igual?
   ├─ Caos? → NÃO FAÇA
   └─ Melhor mundo? → OK fazer

5. Alguém se sente explorado?
   ├─ Sim? → Mudar antes
   └─ Não? → OK fazer
```

**Resultado:**

- 5/5 "OKs" → Fazer com confiança
- 4-3 "OKs" → Consultar CEO + comité
- 2-0 "OKs" → NÃO FAZER

### Casos Reais

#### Caso 1: Cliente quer "falseta" no relatório

```
Cliente: "Aumenta números no relatório, ninguém vai ver"
Resposta: "Não. Seus dados são você. Número falso = decisão ruim."

Por quê?
├─ Cliente é prejudicado (decision-makers enganados)
└─ Nós ficamos com risco legal (fraude)

Alternativa: "Quer entender por que número é baixo? Ajudamos."
```

#### Caso 2: Fornecedor caro vs concorrente

```
Fornecedor A: R$ 1000/mês (qualidade ⭐⭐⭐⭐⭐)
Fornecedor B: R$ 500/mês (qualidade ⭐⭐)

Opção 1: Trocar pra B (economizar)
└─ Clientes sofrem com qualidade ruim
└─ Reputação cai
└─ Economia = falsa

Opção 2: Manter A + aumentar preço
└─ Transparente: "Melhor qualidade = custa mais"
└─ Clientes escolhem se vale
└─ Honestidade

Resposta: Opção 2

Caso especial: "B é 80% tão bom mas R$200"
├─ Usar B pra casos menores
├─ Transparência: "Pra X, usamos fornecedor Y que é ótima relação preço"
└─ Cliente escolhe
```

#### Caso 3: Competidor copiou nossa ideia

```
Opção 1: Copiar dele também (retaliação)
└─ Temos razão
└─ Mas 2 erros ≠ 1 certo

Opção 2: Ação legal (processo)
├─ Se violação clara, caro mas justo
├─ Se grey area, caro + risco

Opção 3: Inovar (ficar à frente)
├─ Fazer MELHOR que cópia
├─ Focus em diferencial
├─ Deixar para trás

Resposta: Opção 3 (+ Opção 2 só se violação clara)
```

## 📢 Denúncia de Violações Éticas

### Como Reportar

```
1. Conversação amigável
   └─ Direto com pessoa: "Acho que errou..."

2. Escalação interna
   └─ Email: conduct@tupansoft.com.br

3. Denúncia anônima
   └─ https://tupansoft.com.br/ethics-report (encrypted)

4. Whistleblower externo
   └─ Conselho ou advogado
```

### Proteção de Denunciantes

```
✅ GARANTIMOS:
├─ Sem retaliação (lei Federal)
├─ Anonimato (se quiser)
├─ Investigação séria
├─ Feedback do resultado
└─ Suporte psicológico (se necessário)

❌ NÃO TOLERAMOS:
├─ "Punir" quem reportou
├─ Demissão por denúncia
├─ Retalho sutil (redução benefício)
└─ Isolamento social
```
